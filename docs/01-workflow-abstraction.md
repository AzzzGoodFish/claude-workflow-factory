# 工作流抽象设计理念

## 概述

工作流是一种将复杂任务分解为可管理单元并按规则执行的编排系统。本文档定义了工作流的四个核心要素：**技能**、**契约**、**上下文**、**节点**、**流程**。

## 核心要素

### 1. 技能 (Skill)

**定义**：领域知识的封装，用于辅助执行特定类型的任务。

**特征**：
- **专注性**：每个技能解决一类特定问题
- **可组合性**：多个技能可组合使用
- **声明式引用**：节点声明所需技能，执行时自动加载

**作用**：为执行者提供"如何做好这件事"的知识指导。

---

### 2. 契约 (Contract)

**定义**：数据规范，约束节点间的输入输出格式和验证规则。

**特征**：
- **强制性**：契约是可执行的验证逻辑，而非仅是文档
- **双向约束**：同时约束生产者（输出）和消费者（输入）
- **可验证性**：每次数据传递都应通过契约验证

**组成**：
- **Schema**：数据结构定义
- **Validator**：验证逻辑
- **Example**：示例数据

**作用**：确保节点间数据传递的正确性，是多智能体协作的基础。

---

### 3. 上下文 (Context)

**定义**：工作流的运行环境，承载状态和数据。

**层级**：
| 层级 | 生命周期 | 用途 |
|------|----------|------|
| 全局上下文 | 跨工作流 | 环境变量、全局配置 |
| 工作流上下文 | 单次执行 | 工作流状态、共享数据 |
| 节点上下文 | 单个节点 | 节点局部状态 |

**载体**：
- **文件系统**：持久化数据、中间产物、最终输出
- **状态对象**：执行状态、进度信息
- **智能体上下文**：对话历史、推理过程（受限于智能体边界）

**作用**：提供数据共享机制，支持状态持久化和恢复。

---

### 4. 节点 (Node)

**定义**：工作流的基本执行单元，负责具体任务的执行。

**特征**：
- **单一职责**：一个节点完成一个明确的任务
- **契约遵守**：严格按照契约消费输入、生产输出
- **技能应用**：根据引用的技能执行任务

**作用**：实现工作流的具体功能。

---

### 5. 流程 (Flow)

**定义**：节点的组织和调度规则。

**控制结构**：
| 结构 | 说明 |
|------|------|
| 顺序 | 节点按序执行，前序完成后执行后序 |
| 并行 | 多个独立节点同时执行 |
| 条件 | 根据结果选择分支 |
| 循环 | 重复执行直到满足条件 |
| 异常处理 | 失败时的重试、回退、上报 |
| 人工介入 | 暂停等待用户确认或输入 |

**作用**：定义工作流的执行逻辑和控制流。

---

## 要素间的关系

```
┌─────────────────────────────────────────────────────────┐
│                        流程 (Flow)                       │
│              定义节点如何组织和调度                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────┐    契约     ┌─────────┐    契约           │
│   │  节点A  │ ─────────→ │  节点B  │ ─────────→ ...    │
│   └────┬────┘   (验证)    └────┬────┘   (验证)          │
│        │                       │                        │
│        ↓                       ↓                        │
│   ┌─────────┐             ┌─────────┐                   │
│   │ 技能 1  │             │ 技能 2  │                   │
│   │ 技能 2  │             │ 技能 3  │                   │
│   └─────────┘             └─────────┘                   │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                      上下文 (Context)                    │
│              文件系统 / 状态对象 / 共享数据                │
└─────────────────────────────────────────────────────────┘
```

**解读**：
- **流程**编排**节点**的执行顺序
- **节点**引用**技能**获取领域知识
- **节点**间通过**契约**约束数据传递
- **上下文**作为底层环境，承载所有运行时状态和数据
