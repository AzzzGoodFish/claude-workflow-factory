# Claude Code 能力与工作流映射

## 概述

本文档建立工作流抽象概念与 Claude Code 能力的对应关系，说明每种能力在工作流中承担的角色及原因。

## 能力概览

| 能力 | 本质 |
|------|------|
| Skill | 可复用的领域知识 |
| Command | 工作流触发入口，注入给主调度的提示词，描述工作流如何执行 |
| Subagent | 独立执行任务的子智能体，每个节点对应一个 subagent |
| Hook | 事件驱动的拦截与处理，用于校验、转换、拦截数据，记录工作流执行状态 |
| Tool/MCP | 执行具体操作的工具，Tool 为 claude code 内置，MCP 为可扩展的外部工具 |
| AskUserQuestion | 同步的用户交互 |
| 文件系统 | 持久化的数据载体 |

## 能力角色定位

### Skill

**是什么**：封装领域知识的可复用模块，自动激活，渐进式披露，可集成脚本。

**工作流角色**：技能载体。

**为什么**：
- 可复用性：同一 Skill 可被多个节点引用
- 渐进式披露：一个 Skill 可适配不同场景（语言、框架）
- 解耦：知识与执行分离，便于维护

### Command

**是什么**：用户显式调用的指令，接收参数，向 Claude 注入上下文。

**工作流角色**：工作流入口 + 主调度器。

**为什么**：
- 入口：自然的用户交互起点，接收工作流参数
- 调度：主智能体执行 Command，可决策节点顺序、调用 Subagent、维护状态
- 全局视角：持有工作流定义，掌控全局流程

### Subagent

**是什么**：通过 Task 工具调用的独立智能体，有独立上下文和工具集。

**工作流角色**：节点执行者。

**为什么**：
- 专注性：绑定特定 Skill，专注单一任务
- 隔离性：独立上下文，避免污染主调度
- 规范性：通过契约检查输入，约束输出，确保数据规范
- 可约束：可限定工具集，实现权限管理
- 无状态：接收输入，产出输出，不关心全局状态

### Hook

**是什么**：事件驱动的拦截机制，在工具调用前后自动触发。

**工作流角色**：契约校验 + 数据转换 + 状态治理 + 流程守护。

**为什么**：
- 不可见性：智能体无感知，适合做基础设施
- 强制性：自动触发，确保契约必被校验
- 灵活性：支持 prompt（智能体语义校验）和 command（脚本校验）两种模式
- 可扩展：可实现数据转换、日志记录、安全拦截等

**状态治理**：通过 Hook 拦截 Task 工具调用，可自动记录节点执行状态：
- PreToolUse：通过 transcript_path，处理 claude code 的 jsonl 日志，识别 Subagent 名称，记录"开始执行"
- PostToolUse：通过 transcript_path，处理 claude code 的 jsonl 日志，识别 Subagent 名称和执行结果，记录"完成/失败"
- 主调度无需手动维护状态，实现执行状态的自动化治理

### Tool / MCP

**是什么**：执行具体操作的能力单元。Tool 为内置，MCP 为可扩展。

**工作流角色**：执行工具。

**为什么**：
- Tool 是 claude code 内置的基础能力（文件、命令等）
- MCP 是外部引入的可扩展的高级能力（截图、浏览器、外部服务等）
- 通过限定 Subagent 可用工具，实现权限管理

### AskUserQuestion

**是什么**：向用户提问并等待响应的工具。

**工作流角色**：同步用户交互。

**为什么**：
- 阻塞式：暂停流程，等待用户决策
- 结构化：支持选项和开放式问题
- 适用于用户在场的交互场景

**补充**：对于长时间无人值守的工作流，可通过 Skill 脚本实现异步通知（如系统提示、短信）。

### 文件系统

**是什么**：持久化的数据存储。

**工作流角色**：上下文载体 + 状态存储。

**为什么**：
- 持久化：支持断点续传
- 可追溯：每个节点的输入输出可查
- 共享机制：节点间通过文件传递数据，符合契约验证
- 状态管理：工作流执行状态可落地为结构化文件

## 映射总结

| 工作流要素 | Claude Code 能力 | 说明 |
|-----------|-----------------|------|
| **技能** | Skill | 直接对应 |
| **契约** | Hook + Schema 文件 | Hook 执行校验，Schema 定义规范 |
| **上下文** | 文件系统 | 节点产物、共享数据、状态存储 |
| **节点** | Subagent | 绑定技能，引用契约，执行具体任务 |
| **流程-入口** | Command | 接收参数，启动工作流 |
| **流程-调度** | Command (主智能体) | 决策节点顺序，调用 Subagent |
| **流程-执行** | Subagent | 执行具体节点任务 |
| **流程-交互** | AskUserQuestion / Skill 脚本 | 使用 claude code 请求用户交互或利用脚本通知用户 |
| **流程-守护** | Hook | 契约校验、数据转换、记录工作流执行状态 |
