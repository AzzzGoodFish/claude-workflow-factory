# cc-wf 工作流概览

## 1. 什么是 cc-wf

cc-wf (Claude Code Workflow) 是基于 Claude Code 原生能力构建的工作流框架，将业务流程抽象为可编排的节点序列。

```
┌─────────────────────────────────────────────────────────────┐
│                        cc-wf 核心概念                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   用户输入  ──→  [入口]  ──→  [节点1] ──→ [节点2] ──→  输出  │
│                   │           │   ↑       │   ↑            │
│                   │           │   │       │   │            │
│                   ▼           ▼   │       ▼   │            │
│               调度指令      技能  契约   技能  契约          │
│                                                             │
└─────────────────────────────────────────────────────────────┘

映射关系：
  ├── 入口 (wf-entry)    →  Claude Code Command
  ├── 节点 (wf-node)     →  Claude Code Subagent
  ├── 技能 (wf-skill)    →  Claude Code Skill
  └── 契约 (contract)    →  Hook Script 校验
```

---

## 2. 组件映射

| cc-wf 概念 | Claude Code 实现 | 职责 |
|-----------|-----------------|------|
| **wf-entry** | Command (.md) | 工作流入口，调度节点执行顺序 |
| **wf-node** | Subagent (.md) | 执行单一任务，绑定技能和契约 |
| **wf-skill** | Skill (SKILL.md) | 领域知识，注入节点上下文 |
| **contract** | Schema + Hook | 节点输入输出规范和校验 |

## 3. 运行时数据流

```
┌────────────────────────────────────────────────────────────────────┐
│                           运行时架构                                │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  用户: /my-workflow arg1 arg2                                      │
│         │                                                          │
│         ▼                                                          │
│  ┌─────────────────┐                                               │
│  │   wf-entry      │  ← Command 加载                               │
│  │   (主调度器)     │  ← 通过 '!' 和 '$ARGUMENTS'  自动写入参数      │
│  └────────┬────────┘                                               │
│           │                                                        │
│           │  !`mkdir -p .context/outputs`                       │
│           │  !`echo '$ARGUMENTS' > .context/params.json`        │
│           │                                                        │
│           ▼                                                        │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  .context/                                                  │   │
│  │  ├── params.json          ← 工作流参数                       │   │
│  │  ├── state.md             ← 执行状态 (wf-state.py 维护)      │   │
│  │  └── outputs/                                                │   │
│  │      ├── node-a.json      ← 节点 A 输出                      │   │
│  │      └── node-b.json      ← 节点 B 输出                      │   │
│  └─────────────────────────────────────────────────────────────┘   │
│           │                                                        │
│           │  Task(node-a, prompt="执行节点 A")                     │
│           ▼                                                        │
│  ┌─────────────────┐     读取      ┌─────────────────┐            │
│  │   wf-node-a     │ ───────────→  │  params.json    │            │
│  │   (Subagent)    │               └─────────────────┘            │
│  │                 │                                               │
│  │  skills: [...]  │  ← 技能内容注入上下文                         │
│  │                 │                                               │
│  │  hooks:         │                                               │
│  │    Stop: [...]  │  ← 输出契约校验                               │
│  └────────┬────────┘                                               │
│           │                                                        │
│           │  节点输出到 transcript（不直接写文件）                   │
│           ▼                                                        │
│  ┌─────────────────┐                                               │
│  │ contract-       │  ← Stop hook 触发（节点内）                    │
│  │ validator.py    │     校验输出是否符合契约                       │
│  └────────┬────────┘                                               │
│           │                                                        │
│           │  校验通过 → 节点完成                                    │
│           │  校验失败 → 阻止退出，要求修正                          │
│           ▼                                                        │
│  ┌─────────────────┐                                               │
│  │ wf-state.py     │  ← PostToolUse hook 触发（节点外）             │
│  │                 │     1. 记录节点完成状态                        │
│  │                 │     2. 从 stdin 的 tool_response 获取输出      │
│  │                 │     3. 写入 .context/outputs/node-a.json       │
│  │                 │     4. 可选：生成 .md 便于后续节点阅读         │
│  └────────┬────────┘                                               │
│           ▼                                                        │
│  ┌─────────────────┐                                               │
│  │   wf-entry      │  ← 继续调度下一节点                           │
│  │   (主调度器)     │                                               │
│  └────────┬────────┘                                               │
│           │                                                        │
│           │  Task(node-b, prompt="执行节点 B")                     │
│           ▼                                                        │
│  ┌─────────────────┐     读取      ┌─────────────────┐            │
│  │   wf-node-b     │ ───────────→  │  node-a.md      │            │
│  │   (Subagent)    │               │  (前序输出,可读) │            │
│  │                 │               └─────────────────┘            │
│  │  系统提示中包含  │                                               │
│  │  数据引用路径    │                                               │
│  └─────────────────┘                                               │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

## 4. 节点间数据传递

```
┌─────────────────────────────────────────────────────────────┐
│                   节点输出写入机制                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  节点 A 完成后：                                            │
│                                                             │
│    node-a 输出                                              │
│        │                                                    │
│        │  (输出到 transcript，不直接写文件)                 │
│        ▼                                                    │
│    Stop hook (contract-validator.py)                        │
│        │  校验输出符合契约                                  │
│        ▼                                                    │
│    PostToolUse hook (wf-state.py)                           │
│        │                                                    │
│        ├─→ 从 stdin 的 tool_response 直接获取输出           │
│        │   （无需读取 transcript）                          │
│        │                                                    │
│        ├─→ 写入 .context/outputs/node-a.json (原始)         │
│        │                                                    │
│        └─→ 写入 .context/outputs/node-a.md   (可读版)       │
│                                                             │
│  💡 节点只负责产出输出，不关心持久化路径                     │
│  💡 wf-state.py 统一处理输出提取和格式转换                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                   节点输入读取机制                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  节点 B 的系统提示 (agents/node-b.md):                      │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ ## 数据读取                                           │  │
│  │                                                       │  │
│  │ 在开始工作前，读取以下数据：                           │  │
│  │                                                       │  │
│  │ - **工作流参数**: .context/params.md               │  │
│  │ - **节点 A 输出**: .context/outputs/node-a.md      │  │
│  │                                                       │  │
│  │ > 使用 Read 工具读取上述文件后再执行业务逻辑。         │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                             │
│  执行流程：                                                  │
│                                                             │
│    wf-entry                                                 │
│        │                                                    │
│        │ Task(node-b, prompt="执行节点 B")                  │
│        ▼                                                    │
│    node-b 启动                                              │
│        │                                                    │
│        │ Read(.context/params.md)                        │
│        │ Read(.context/outputs/node-a.md)  ← 读前序输出  │
│        ▼                                                    │
│    执行业务逻辑                                              │
│        │                                                    │
│        │ 输出到 transcript（不写文件）                      │
│        ▼                                                    │
│    Stop hook → 契约校验 → PostToolUse → wf-state.py 写文件  │
│                                                             │
│  💡 主调度不传递业务数据，只发调度指令                       │
│  💡 节点读取 .md 格式，更易理解上下文                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 7. 输出目录结构

```
.claude/
├── commands/
│   └── {workflow-name}.md      ← wf-entry (主调度)
│
├── agents/
│   ├── {node-a}.md             ← wf-node (节点)
│   └── {node-b}.md
│
├── skills/
│   └── {skill-name}/
│       └── SKILL.md            ← wf-skill (技能)
│
├── contracts/
│   └── {contract-name}.yaml    ← 契约定义
│
├── hooks/
│   ├── contract-validator.py   ← 契约校验脚本
│   └── wf-state.py             ← 状态治理脚本
│
└── settings.json               ← 全局配置 (MCP, 全局 hooks)


.context/                       ← 运行时数据
├── params.json                 ← 工作流参数
├── state.md                    ← 执行状态
└── outputs/
    ├── {node-name}.json        ← 节点输出（原始）
    └── {node-name}.md          ← 节点输出（可读）
```

## 9. 关键技术点速查

| 问题 | 解决方案 |
|------|----------|
| 节点如何获取输入？ | 从 `.context/outputs/prev-node.md` 读取（可读格式） |
| 节点输出如何持久化？ | wf-state.py 在 PostToolUse 时从 tool_response 提取并写入 |
| 为什么用 .md 格式？ | agent 阅读 markdown 更友好，便于理解上下文 |
| 契约名称如何传递？ | node-builder 生成时写入 hook command 参数 |
| subagent_type 如何获取？ | prompt 匹配：从 agent_transcript 获取 prompt，在主会话中匹配 Task |
| 节点输出如何校验？ | Stop hook (自动转 SubagentStop) 触发 contract-validator.py |
| 并发节点如何执行？ | 同一消息中发起多个 Task 调用 |
| 断点如何续传？ | 读取 state.md 中的 session_id，恢复会话 |
